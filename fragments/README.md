# Вебинар 15. Фрагменты

1. Фрагмент - компонент пользовательского интерфейса.
2. FragmentManager - управление фрагментами.
3. Повторное использование фрагментов для разных типов устройств.
4. Использование функций обратного вызова для взаимодействия между фрагментами.
5. Передача параметров во фрагмент и обратно.
6. Использование общих моделей для связи фрагментов.

## Вопросы для самопроверки:

1. Что такое фрагмент?
   [] Фрагмент - это отдельный поток выполнения в Android приложении.
   [] Фрагмент - это часть пользовательского интерфейса Activity, которая имеет свой собственный жизненный цикл и может быть переиспользована в разных Activity.
   [] Фрагмент - это компонент, который отвечает за фоновые операции.
2. Чем фрагмент отличается от собственного View?
   [] Фрагмент всегда отображается на весь экран, в то время как View может быть любого размера.
   [] Фрагмент имеет собственный жизненный цикл, который тесно связан с жизненным циклом Activity, и может содержать несколько View. Собственный View - это 
      единичный UI-элемент, который не имеет своего жизненного цикла, а только жизненный цикл родительского View или Activity.
   [] Фрагмент не может быть добавлен в макет XML, а View может.
3. Как управлять фрагментами?
   [] Управление фрагментами осуществляется с помощью `Context` напрямую.
   [] Управление фрагментами осуществляется с помощью `FragmentManager`, который позволяет добавлять, удалять, заменять и взаимодействовать с фрагментами 
      в Activity.
   [] Управление фрагментами происходит автоматически системой Android без вмешательства разработчика.
4. Как передать параметры во фрагмент?
   [] Параметры передаются во фрагмент с использованием глобальных переменных.
   [] Параметры передаются во фрагмент с использованием `Bundle` и метода `setArguments()`. В конструкторе фрагмента можно получить эти аргументы.
   [] Параметры передаются во фрагмент напрямую через его конструктор.
5. Как получить результат из фрагмента?
   [] Результат из фрагмента всегда возвращается через `Intent` в `onActivityResult()` родительской Activity.
   [] Результат из фрагмента может быть получен с использованием `Fragment Result API` (установка слушателя результата в родительском компоненте и установка 
      результата во фрагменте), общих `ViewModel`, или через колбэки (интерфейсы), реализованные в родительской Activity/Fragment.
   [] Результат из фрагмента нельзя получить, так как фрагмент не возвращает никаких значений.
6. Как использовать общие модели для связи фрагментов?
   [] Общие модели используются для хранения данных, которые будут доступны только одному фрагменту.
   [] Общие модели (ViewModel), привязанные к жизненному циклу Activity или родительского фрагмента, позволяют нескольким фрагментам получать доступ к одним 
      и тем же данным и обмениваться ими. Изменения в ViewModel автоматически отображаются во всех подписанных фрагментах.
   [] Общие модели используются для создания статических методов, доступных всем фрагментам.
