# Вебинар 23.1. DataStore

1. Использование DataStore в приложении против собственных решений
2. Хранение данных ключ-значение: Preferences DataStore
3. Хранение данных в виде объектов: Proto DataStore и Protocol Buffers
4. DataStore и пользовательская сериализация - JSON DataStore
5. Шифрование данных в DataStore

## Вопросы для самопроверки:

1. Какие преимущества дает использование DataStore перед собственными решениями?
   [] DataStore обеспечивает полностью синхронную работу с данными, что упрощает отладку.
   [] DataStore основан на Kotlin Coroutines и Flow, предоставляя асинхронный и транзакционный API, что делает его безопасным и производительным, а также 
      устойчивым к исключениям времени выполнения и обеспечивающим атомарные операции.
   [] DataStore позволяет хранить неограниченный объем данных, не зависящий от ресурсов устройства.
2. Какие типы DataStore существуют и для чего они предназначены?
   [] Существуют SQL DataStore для реляционных данных и NoSQL DataStore для нереляционных.
   [] Существуют Preferences DataStore для хранения данных ключ-значение (аналог SharedPreferences, но асинхронный и безопасный) и Proto DataStore для хранения 
      типизированных объектов с использованием Protocol Buffers (требует определения схемы данных).
   [] Существуют Memory DataStore для временного хранения данных и Disk DataStore для постоянного хранения.
3. В каких случаях удобнее применять Proto DataStore перед Preferences DataStore?
   [] Proto DataStore всегда предпочтительнее, так как он обеспечивает максимальную производительность.
   [] Proto DataStore удобнее применять, когда нужно хранить типизированные данные (объекты) со сложной структурой, обеспечивать схему и строгую типизацию 
      данных, а также когда требуется легкая миграция и совместимость версий данных. Preferences DataStore лучше подходит для простых пар ключ-значение.
   [] Proto DataStore удобнее для хранения данных, которые не должны быть сериализованы.
4. Каким образом реализуется сериализация данных в JSON DataStore?
   [] JSON DataStore не существует как отдельный тип DataStore; пользовательская сериализация в JSON реализуется путем создания собственного `Serializer<T>` 
      и использования его с Proto DataStore. Для сериализации/десериализации JSON обычно используются библиотеки типа `kotlinx.serialization`.
   [] JSON DataStore использует встроенный механизм Android для автоматической сериализации любых объектов в JSON.
   [] Сериализация данных в JSON DataStore происходит путем сохранения данных в виде строк, а затем ручного парсинга JSON.
5. Как защитить данные в DataStore от несанкционированного доступа?
   [] Данные в DataStore защищены по умолчанию, никаких дополнительных действий не требуется.
   [] Защитить данные в DataStore от несанкционированного доступа можно добавлением шифрования данных во время их сериализации.
   [] Для защиты данных в DataStore достаточно установить пароль на приложение.
