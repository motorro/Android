## Вопросы для самопроверки:

1. Каковы три основных инструмента Android-разработчика?
   [] Android Studio, Gradle, Эмулятор/Физическое устройство (для запуска).
   [] Android SDK, Java/Kotlin, Git.
   [] Figma, Jira, VS Code.
2. Как происходит сборка Android-приложения?
   [] Сборка Android-приложения происходит с помощью системы автоматизации сборки Gradle, которая компилирует исходный код, ресурсы и манифест в APK-файл.
   [] Приложение собирается автоматически при сохранении любого файла в проекте.
   [] Разработчик вручную компилирует каждый файл и объединяет их в единый исполняемый файл.
3. Что такое зависимости и как они подключаются к проекту?
   [] Зависимости - это внешние библиотеки или модули, которые предоставляет дополнительную функциональность. Они подключаются к проекту путем добавления 
      соответствующих записей в файл build.gradle (или build.gradle.kts) модуля.
   [] Зависимости - это ошибки в коде, которые необходимо исправить перед сборкой приложения.
   [] Зависимости - это набор изображений и медиафайлов, которые автоматически загружаются в проект.
4. Что определяет требования и состав приложения?
   [] Требования и состав приложения определяются файлом AndroidManifest.xml, который описывает компоненты приложения, разрешения и аппаратные функции, 
      необходимые для его работы.
   [] Требования и состав приложения определяются только набором используемых библиотек.
   [] Требования и состав приложения определяются исключительно дизайном пользовательского интерфейса.
5. Какие преимущества дает использование ресурсов?
   [] Использование ресурсов (строки, изображения, макеты и т.д.) позволяет легко адаптировать приложение под разные языки, ориентации экрана и плотности 
      пикселей, а также упрощает поддержку и изменение внешнего вида без изменения кода.
   [] Ресурсы замедляют работу приложения и увеличивают его размер, поэтому их следует избегать.
   [] Ресурсы используются исключительно для хранения конфиденциальных данных приложения.
