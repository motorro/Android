## Вопросы для самопроверки:

1. Каковы три основных инструмента Android-разработчика?
   [] Android Studio, Gradle, Эмулятор/Физическое устройство (для запуска).
   [] Android SDK, Java/Kotlin, Git.
   [] Figma, Jira, VS Code.
2. Как происходит сборка Android-приложения?
   [] Сборка Android-приложения происходит с помощью системы автоматизации сборки Gradle, которая компилирует исходный код, ресурсы и манифест в APK-файл.
   [] Приложение собирается автоматически при сохранении любого файла в проекте.
   [] Разработчик вручную компилирует каждый файл и объединяет их в единый исполняемый файл.
3. Что такое зависимости и как они подключаются к проекту?
   [] Зависимости - это внешние библиотеки или модули, которые предоставляет дополнительную функциональность. Они подключаются к проекту путем добавления 
      соответствующих записей в файл build.gradle (или build.gradle.kts) модуля.
   [] Зависимости - это ошибки в коде, которые необходимо исправить перед сборкой приложения.
   [] Зависимости - это набор изображений и медиафайлов, которые автоматически загружаются в проект.
4. Что определяет требования и состав приложения?
   [] Требования и состав приложения определяются файлом AndroidManifest.xml, который описывает компоненты приложения, разрешения и аппаратные функции, 
      необходимые для его работы.
   [] Требования и состав приложения определяются только набором используемых библиотек.
   [] Требования и состав приложения определяются исключительно дизайном пользовательского интерфейса.
5. Какие преимущества дает использование ресурсов?
   [] Использование ресурсов (строки, изображения, макеты и т.д.) позволяет легко адаптировать приложение под разные языки, ориентации экрана и плотности 
      пикселей, а также упрощает поддержку и изменение внешнего вида без изменения кода.
   [] Ресурсы замедляют работу приложения и увеличивают его размер, поэтому их следует избегать.
   [] Ресурсы используются исключительно для хранения конфиденциальных данных приложения.

## Self-check questions:

1.  What are the three main tools of an Android developer?
    [] Android Studio, Gradle, Emulator/Physical device (for running).
    [] Android SDK, Java/Kotlin, Git.
    [] Figma, Jira, VS Code.
2.  How is an Android application built?
    [] An Android application is built using the Gradle build automation system, which compiles the source code, resources, and manifest into an APK file.
    [] The application is built automatically when any file in the project is saved.
    [] The developer manually compiles each file and combines them into a single executable file.
3.  What are dependencies and how are they connected to the project?
    [] Dependencies are external libraries or modules that provide additional functionality. They are connected to the project by adding corresponding entries 
       to the build.gradle (or build.gradle.kts) file of the module.
    [] Dependencies are errors in the code that need to be fixed before building the application.
    [] Dependencies are a set of images and media files that are automatically downloaded into the project.
4.  What determines the requirements and composition of an application?
    [] The requirements and composition of an application are determined by the AndroidManifest.xml file, which describes the application's components, 
       permissions, and hardware features required for its operation.
    [] The requirements and composition of an application are determined only by the set of libraries used.
    [] The requirements and composition of an application are determined solely by the user interface design.
5.  What are the advantages of using resources?
    [] Using resources (strings, images, layouts, etc.) makes it easy to adapt the application to different languages, screen orientations, and pixel densities, 
       and also simplifies maintenance and changing the appearance without changing the code.
    [] Resources slow down the application and increase its size, so they should be avoided.
    [] Resources are used exclusively for storing confidential application data.
